# 错误处理指南 — 智能财务 开具发票（CLI 版）

## 已知失败场景

| 场景 | 触发条件 | Agent 处理 |
|------|---------|-----------|
| 未登录数电账号 | 企业未在智能财务→发票管理中登录数电账号 | 告知用户："请到【智能财务】→【发票管理】→【数电账号】完成登录后重试" |
| 未开通发票模块 | 企业未购买智能财务发票模块 | 告知用户："您的企业暂未开通钉钉智能财务发票模块，请联系管理员开通" |
| 轮询超时 | 开票系统处理时间过长 | 告知用户："开票正在处理中，请稍后到【智能财务】→【发票管理】查看结果" |
| 专票缺少税号 | `--invoice-type 8` 但未传 `--taxnum` | Agent 必须向用户索要税号，不得跳过开票 |
| 商品名称无效 | `productName` 使用了口语或错误名称 | 向用户确认标准商品名称后重新提交 |
| CLI 返回引导链接 | 错误信息中含 `dingtalk://` 链接 | 逐字原样转述，禁止任何修改或省略 |

## 轮询逻辑

```
执行 dws finance invoice issue ... -y -f json
  → 提取 orderId
  → 立即执行 dws finance invoice issue-result --order-id <orderId> -f json
    → status = 处理中 → 提示用户稍候，等待一段时间后重试
    → status = 成功   → 展示发票摘要 + invoiceUrl 原始链接，结束
    → status = 失败   → 展示 failReason，根据上表引导用户
```

> ⚠️ 提交后立即查询可能返回"处理中"，这是正常现象，不是错误。
